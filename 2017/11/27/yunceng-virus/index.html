灵魂无处安放,所以选择流浪

新型病毒之云层病毒构想

本文总共3281个字 | 您是第位看到它们的小伙伴

文章在先知社区发表，因为敏感原因，屏蔽了很多关键字，导致不怎么完整，这篇文章就是将病毒的，但是也可以说是后门，这个新型病毒的构想在理论上是可以实现的，只是代码构思可能会比较复杂，需要考虑的东西有很多，未来的攻防会很激烈，当然也会更加高端。

0x00 开篇

安全技术存在的价值在于较量,在攻与防中寻求突破一直是技术提升的保障。
首先，介绍一下云层病毒的概念。
云层病毒：这是一种可以寄生在任何文件里的一段数据,它本身不是病毒，不具有危害性，但当它进行代码整合时,就成为了具有破坏性的病毒程序,缘由它自我分解和组合能力和云层的拼接差不多,所以取名为云层病毒。

0x01 云层病毒原理!

当病毒主程序在本地机器上运行时,主程序把病毒核心文件数据分解到任何文件里,然后执行PE头注入,源文件头EIP指向病毒执行代码地址，当任意被感染的文件里存在病毒主程序代码时，开始执行病毒数据组合,之后开始执行病毒使命。

感染文件的基本步骤：

1．判断目标文件开始的两个字节是否为“MZ”。
2．判断PE文件标记“PE”。
3．判断感染标记，如果已被感染过则跳出继续执行程序，休眠2秒，否则进行感染。
4．对标记为PE文件的进行PE头注入感染。
5．主程序对相关病毒核心代码进行分解附加到任何文件末尾,标志位感染标号。
6．主程序任务完成,删除自身,病毒等待触发。

病毒促发的基本步骤：

1.病毒促发,
2.开始组合代码
3.执行病毒

云层病毒加载原理:

当病毒把自身loader注入完并删除后，进入潜伏或者开始活动，在接收完相关数据时，开始执行自身使命，当PE文件被调用或者被执行时，PE文件EIP的地址是指向病毒数据的，这个有点像程序加壳,在被加载进内存时，控制权交给源程序，自身独立出来， 遍历磁盘被感染文件组合核心程序，利用反射dll注入技术注入进程，迁移至系统进程，执行完后退出。

如我这一篇文章（Shellcode的另类使用方式）,其原理也是接收和加载一段shellcode，然后loader源程序控制权交给shellcode执行，shellcode不以文件形式存在。

这种做法其保证了自身只在内存中运行，从而减少被发现的风险。

0x03 云层病毒特点

1.不在磁盘上存在任何文件,以寄生为主。
2.病毒使用反射dll注入技术。
3.只在内存中执行,不产生文件。
4.接收病毒数据方式多样性。
5.变种简单。

云层病毒可以随意的把文件数据写入其他文件的空洞区域（00字节的空白区域），从而保持源文件的大小不变，更好的隐藏，只要写入字节数控制的好，它可以按照编写者的任何思路走，一个文件既可以是病毒寄生数据，也可以是病毒loader,还可以是两者合一，病毒数据接收方式多样性，免杀全球绝对是没有问题的。

0x04 云层病毒利用场景

1.远程利用

1.病毒主文件负责请求外部服务器数据到本机,并把相关数据存储在变量缓冲区中,当下载完毕后，从内存变量中读取数据,附加到任意文件中，注入病毒loader代码到PE文件,删除自身。

2.本地连接远程服务器,远程服务器通过发送数据到本地,本地把相关数据存储在变量缓冲区中,当接收完毕后，从内存变量中读取数据,附加到任意文件中,注入病毒loader代码到PE文件,删除自身。

3.本地连接远程服务器,远程服务器通过发送数据到本地,本地把相关数据进行分段存储,当接收到感染指令的时候,对病毒进行组装并加载进内存,对本地进行感染,对外进行感染,否则进行潜伏。

4.本地连接远程Web服务器,读取远程Web服务器网页上的数据文本,本地把相关数据进行分段存储,注入病毒loader代码到PE文件,删除自身。

2.本地使用

本地执行病毒主程序时,开始注入PE文件,修改EIP指向,分解病毒程序,删除自身。

3.后门使用

本地执行病毒主程序时,开始注入PE文件,修改EIP指向,分解病毒程序,删除自身。
当病毒被激活时,主动或者被动监听端口,等待接收Shellcode并执行命令。

0x05 云层病毒简单原理演示

病毒程序：一个弹MessageBox的程序。
感染程序：cmd.exe
这里我就不演示病毒loader了，就简单演示病毒怎么组合代码，为了易懂，我们用python代码。
我们把病毒程序（MessageBox.exe）以十六进制读取出来,保存为messagebox.txt。

读取代码

#! /usr/bin/env python
# -*- coding: utf-8 -*-

import os
import sys

def payload(files):
    shellcode = ""
    ctr = 1
    maxlen = 15 #to create rows
    try:
        for b in open(files, "rb").read():
            shellcode +=b.encode("hex")
            if ctr == maxlen:
                ctr = 0
            ctr += 1
        print "Code length: " + str(len(shellcode))
        return shellcode
    except:
        print "转换失败，请检查!"

f = open('messagebox.txt','w')
files = "messagebox.exe" 
a = payload(files)
f.write(a)
f.close()

然后进行代码分解追加代码到文件中,这里代码少，不做分解。
然后查看数据，病毒数据已全部分解注入到目标程序了，并且程序可正常运行。

分解注入代码

f1 = open('cmd.exe','a+')
F2= open('messagebox.txt','r')

下面就是从文件中读取数据进行代码重组了

0x06 结束

本篇只简单对云层病毒进行了一个科普，目前病毒技术千变万化，许多杀毒软件面对新型木马病毒都显得是个摆设，由于有些强力杀毒软件的防御能力和系统自身权限的原因，导致写文件时会促发告警，所以向不是系统的相关文件里面写入病毒loader能够减少被杀的风险。云层病毒应用可以很广,编写思路灵活,所以变种相当容易,被查杀的风险很低，我们使用云层病毒用来做后门其实是相当不错的。
